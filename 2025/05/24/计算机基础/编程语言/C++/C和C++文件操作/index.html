C/C++文件操作

文件到底是什么？

在计算机科学中，文件是存储在持久化介质（如硬盘、SSD、U盘等）上的逻辑数据单元，是操作系统管理数据的基本载体。文件通过名称-内容的二元结构组织信息，为用户和程序提供了一种抽象的数据存取方式，屏蔽了底层物理介质的复杂性。

可以详见参考之前的：外存管理 - The Peak Tower

文件的本质与作用

文件是操作系统对存储设备的抽象映射，它将物理介质的二进制序列转化为可管理的逻辑单元。其核心价值在于：

  • 持久化保存：断电后数据不丢失，区别于内存的临时性存储。
  • 结构化访问：通过文件系统（如NTFS、EXT4）实现按名称寻址、权限控制、读写操作等标准化管理。
  • 资源共享：作为进程间通信的桥梁，允许多个程序读写同一数据源。

文件的组成要素

一个完整的文件由以下部分构成：

元数据（Metadata） 描述文件属性的系统数据，包括：

  • 名称与扩展名（如report.pdf）
  • 存储路径（如/home/user/docs/）
  • 大小、创建/修改时间
  • 权限（读/写/执行）
  • 所有者与用户组（Unix-like系统）

数据内容 用户实际存储的信息，可以是：

  • 文本：ASCII/Unicode编码的字符序列（如.txt文件）
  • 二进制：结构化字节流（如.exe可执行文件）
  • 混合型：包含头部和数据区（如.png图像文件）

文件与内存的关系

文件与内存的交互是计算的核心过程：

  • 加载（Load）：程序运行时，操作系统将可执行文件内容映射到内存（如PE/ELF加载）
  • 映射（mmap）：通过内存映射技术直接关联文件与内存地址，避免复制开销
  • 交换（Swap）：当物理内存不足时，将部分数据暂存到磁盘交换文件

注意下面的关键点：

文件 = 内容 + 属性，对文件的操作就是对文件内容和文件属性的操作。当文件没有被操作的时候，文件一般都是在磁盘上存放。当我们对文件操作时，文件都会被提前加载到内存中，加载的内容至少得有属性。当文件被加载到内存中时，在Linux下并不一定只有你一个人在打开文件，内存中一定存在大量的不同文件的属性。

因此，打开文件的本质就是将文件属性加载到内存中，OS中一定存在大量被打开的文件，操作系统对这些被打开的文件进行管理需要先描述、再组织，所以需要先构建在内存中的文件结构体(struct file)。

文件可以被分为两部分：磁盘文件和被打开的文件(内存文件)。文件是被OS所打开的，是被用户所创建的进程让OS打开的。我们之前所有的文件操作，都是进程(struct task_struct) 和被 打开文件(struct file) 的关系。

文件描述符的理解

进程可以打开多个文件，这也就意味着系统中一定会存在大量的被打开的文件，然而被打开的文件则需要被操作系统管理，我们知道，管理的本质就是先描述在组织，所以操作系统为了管理对应的打开文件，操作系统必定要为文件创建对应的内核数据结构来标识文件，这个内核数据结构就是struct file{}结构体（与C语言的FILE没有关系哦）；包含了文件的大部分属性。

在Windows PE格式的可执行文件（.exe）中，数据存储采用小端模式（Little-Endian），而C程序在x86/x64架构的CPU上运行时，默认也使用小端模式。C程序读取PE文件（如.exe）中的数据后，内存中的存储方式仍然是小端（Little-Endian），且可以直接使用而无需转换。

  • 无需手动转换：在小端 CPU（如 x86/x64）上，所有内存地址和整数值的存储默认就是小端模式。
  • 直接使用即可：无论是硬编码的地址、指针运算，还是从内存中读取的地址，CPU 会自动按小端处理。

基本知识

C++ 标准库中提供了 3 个类用于实现文件操作，它们统称为文件流类，这 3 个类分别为：

  • ifstream：专用于从文件中读取数据；
  • ofstream：专用于向文件中写入数据；
  • fstream：既可用于从文件中读取数据，又可用于向文件中写入数据。

以下是C++文件操作三种流类型的对比表格：

类型 功能特点 典型使用场景
ofstream - 专用于文件写入
- 自动创建不存在的文件
- 默认覆盖写入(可设追加模式)
- 支持<<运算符和write()方法		 - 程序日志记录
- 数据报告导出
- 配置文件生成
- 对象序列化存储
ifstream - 专用于文件读取
- 文件必须存在才能打开
- 支持>>运算符和getline()
- 可检测文件结束(eof())		 - 配置文件读取
- 数据文件导入
- 资源文件加载
- 对象反序列化
fstream - 同时支持读写操作
- 需显式指定打开模式
- 需手动控制文件指针位置
- 适合频繁读写切换		 - 数据库文件操作
- 实时数据更新文件
- 大型文件随机访问
- 需要读写交替的工作文件

值得一提的是，这 3 个文件流类都位于 头文件中，因此在使用它们之前，程序中应先引入此头文件。

可以看到，

  • ifstream 类和 fstream 类是从 istream 类派生而来的，因此 ifstream 类拥有 istream 类的全部成员方法。
  • 同样地，ofstream 和 fstream 类也拥有 ostream 类的全部成员方法。

这也就意味着，istream 和 ostream 类提供的供 cin 和 cout 调用的成员方法，也同样适用于文件流。

值得一提的是，和 头文件中定义有 ostream 和 istream 类的对象 cin 和 cout 不同， 头文件中并没有定义可直接使用的 fstream、ifstream 和 ofstream 类对象。因此，如果我们想使用该类操作文件，需要自己创建相应类的对象。

fstream 类拥有 ifstream 和 ofstream 类中所有的成员方法，这 罗列了 fstream 类一些常用的成员方法。

成员方法名 适用类对象 功 能

文件打开模式

模式标记 适用对象 作用
ios::in (i)fstream 打开文件用于读取数据。如果文件不存在，则打开出错。
ios::out (o)fstream 打开文件用于写入数据。如果文件不存在，则新建该文件；如果文件原来就存 在，则打开时清除原来的内容。
ios::app (o)fstream 打开文件，用于在其尾部添加数据。如果文件不存在，则新建该文件。
ios::ate (i)fstream 打开一个已有的文件，并将文件读指针指向文件末尾（读写指 的概念后面解 释）。如果文件不存在，则打开出错。
ios::trunc (o)fstream 打开文件时会清空内部存储的所有数据，单独使用时与 ios::out 相同。
ios::binary (i/o)fstream 以二进制方式打开文件。若不指定此模式，则以文本模式打开。
ios::in | ios::out fstream 打开已存在的文件，既可读取其内容，也可向其写入数据。文件刚打开时，原有内容保持不变。如果文件不存在，则打开出错。
ios::in | ios::out ofstream 打开已存在的文件，可以向其写入数据。文件刚打开时，原有内容保持不变。如果文件不存在，则打开出错。
ios::in | ios::out | ios::trunc fstream 打开文件，既可读取其内容，也可向其写入数据。如果文件本来就存在，则打 开时清除原来的内容；如果文件不存在，则新建该文件。
