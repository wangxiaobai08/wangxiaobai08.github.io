详解linux系统下proc文件夹目录

简单介绍

/proc 是 Linux 系统中一个特殊的虚拟文件系统（procfs），它不占用磁盘空间，而是由内核动态生成，用于提供系统内核和进程的实时信息。用户和程序可以通过读取或修改 /proc 下的文件来获取或调整系统运行状态，由于系统的信息，如进程，是动态改变的，所以用户或应用程序读取proc文件时，proc文件系统是动态从系统内核读出所需信息并提交的。

基于/proc文件系统如上所述的特殊性，其内的文件也常被称作虚拟文件，并具有一些独特的特点。例如，其中有些文件虽然使用查看命令查看时会返回大量信息，但文件本身的大小却会显示为0字节。此外，这些特殊文件中大多数文件的时间及日期属性通常为当前系统时间和日期，这跟它们随时会被刷新（存储于RAM中）有关。

在 Linux 系统中，/proc/[PID]/maps 文件（注意不是目录）记录了 进程的内存映射详情，包括代码段、堆、栈、动态库等内存区域的地址范围、权限和来源文件。

文件格式说明

每行代表一个内存映射区域，格式如下：

（字段用空格分隔，部分字段可能为空）

字段 说明
start-end 内存区域的起始和结束地址（十六进制）。
permissions 权限标志： - r = 可读 - w = 可写 - x = 可执行 - p = 私有（COW） - s = 共享
offset 映射文件在文件中的偏移量（十六进制）。
dev 文件所在设备的设备号（主设备号:次设备号）。
inode 文件的 inode 号（0 表示匿名映射）。
pathname 映射的文件路径（匿名映射显示为 [anon]，堆/栈等有特殊标记）。

区别

在 Linux 的 /proc 文件系统中，/proc/self/task 和 /proc/[PID]/task/ 的核心功能是类似的（均用于查看线程信息），但它们的 指向对象 和 使用场景 有所不同。

路径 动态性 目标进程 权限要求
/proc/self/task 自动指向当前进程 当前进程（自身） 无需额外权限（属于自己）
/proc/[PID]/task/ 固定指向指定 PID 的进程 其他进程（指定PID） 需目标进程的访问权限

其中/proc/self 是一个符号链接，始终指向 当前访问该路径的进程。例如：

  • 进程 A 访问 /proc/self/task → 显示进程 A 的线程信息。
  • 进程 B 访问 /proc/self/task → 显示进程 B 的线程信息。

在程序内部查看自身的线程信息（无需硬编码 PID）。
在调试多线程程序时，直接在代码中读取 /proc/self/task 以分析线程状态。

而**/proc/[PID]/task/通过替换 [PID] 为具体进程 ID，查看 指定进程 的线程信息。用途**是监控或调试其他进程的线程（例如系统工具 top, htop, ps 等）。
