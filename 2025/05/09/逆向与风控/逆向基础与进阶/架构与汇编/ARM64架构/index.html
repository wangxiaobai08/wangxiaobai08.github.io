32位和64位架构的不同？

以下是将所有信息合并后的简洁对比表格：

特征 32位架构（ARM/32位） 64位架构（ARM64/64位）
架构位数 32位，使用ARMv7及之前版本的指令集 64位，使用ARMv8-A及后续版本的指令集
寄存器宽度 32位（4字节），每次操作32位数据 64位（8字节），每次操作64位数据
内存寻址能力 最大支持4GB内存（2^32字节） 理论上支持18.4亿TB内存（2^64字节）
指令集架构（ISA） 32位指令集，指令最大处理32位数据 64位指令集，指令最大处理64位数据
指针大小 4字节（32位指针） 8字节（64位指针）
性能 性能较低，适用于低功耗设备，计算密集型任务有限 高性能，适用于大数据、高计算任务、高并发处理
操作系统支持 只能运行32位操作系统与应用程序 兼容32位应用程序，支持64位操作系统与应用程序
总线宽度 数据传输带宽较低，受限于32位总线宽度 数据传输带宽较高，支持更宽的64位总线宽度
内存占用 由于指针较小，内存占用较少 由于指针占8字节，内存占用更大
兼容性 仅支持32位应用程序和操作系统 兼容32位应用程序，但能充分发挥64位架构的优势（更大内存、更高性能）

2000年以后

进入2000年代，随着移动设备的崛起，ARM架构逐渐成为移动处理器的主导。ARM9和ARM11在处理性能、功耗和多媒体功能方面进行了优化。

2010年代至今，ARM推出了Cortex系列处理器，进一步扩展了架构的性能和能效。其中在Cortex-A系列(用于智能手机、平板电脑和服务器)中ARMv7-A引入了对多核处理的支持，ARMv8-A则引入了64位指令集（AArch64），大大提升了处理器的性能。注意：ARMv8引入的64位指令集支持更大规模的数据处理和更大的地址空间。指令集分为32位的AArch32和64位的AArch64，二者可以混合使用。(现在发售的智能手机一般就是arm64-v8a架构的)

注意：尽管 AArch64 的指令宽度是固定的 32 位，但它在寄存器和数据处理方面支持 64 位操作，这是它与 ARM32 最大的区别之一

在 Android 应用的 APK 文件中 lib 目录下，用于存放专门为各个不同架构编译的本地库（.so 文件）

高版本的 ARM64 架构能运行低版本的 ARM 应用，但为了充分利用设备的硬件性能，推荐使用与设备架构匹配的 64 位库。

ARMv8-A 架构介绍

ARMv8-A 是 ARM 公司发布的第一代支持 64 位处理器的指令集和架构。ARMv8-A 架构除了提高了处理能力外，还引入了很多吸引人的新特性。

  • 具有 64 位宽的虚拟地址空间。
  • 提供 31 个 64 位宽的通用寄存器，可以减少对栈的访问，从而提高性能。
  • 提供 16KB 和 64KB 的 page，有助于降低 TLB 的未命中率（miss rate）。
  • 具有全新的异常处理模型，有助于降低操作系统和虚拟化的实现复杂度。
  • 具有全新的加载-获取、存储-释放指令（load-acquire，store-release instruction），专为 C++11 以及 Java 内存模型而设计。

ARMv8处理器的运行状态

AArch64架构的异常等级确定了处理器当前运行的特权级别，类似于ARMv7架构中的特权等级。一般分为四个等级：

EL0：用户特权，用于运行普通用户程序
EL1：系统特权，通常用于运行操作系统
EL2：运行虚拟化扩展的虚拟监控程序（hypervisor）
EL3：运行安全世界中的安全监视器

寄存器

通用寄存器

ARM64 中有 31 个 64 位的通用寄存器，分别为 X0 到 X30，每个寄存器的大小是 64 位。这些寄存器用于存储数据、指针、地址等信息。

  • X0 到 X7：这些寄存器通常用于函数调用的参数传递和返回值的存储。
  • X8 到 X15：这些寄存器用于临时存储和计算结果，在函数调用时不需要保留它们的值。
  • X16 和 X17：这两个寄存器是特别的“链接寄存器”，用于存储某些特殊的值，常常在系统调用或某些指令（如 MOV 或 BL）中使用。
  • X18 到 X29：这些寄存器主要用于保存临时变量或者系统调用中的一些数据。
  • X30：链接寄存器（LR），通常用于保存函数调用的返回地址。

通用寄存器的别名：

  • W0 到 W30：这些是 X0 到 X30 的低 32 位部分，用于 32 位操作。

特殊寄存器

零寄存器

ARMv8 架构提供两个零寄存器（zero register），这两个零寄存器的内容全是 0，可以用作源寄存器，也可以当作目标寄存器。WZR 是 32 位的零寄存器，XZR 是 64 位的零寄存器。

PC 寄存器

PC （Program Counter）寄存器通常用来指向当前运行指令的下一条指令的地址，用于控制程序中指令的执行顺序，但是编程人员不能通过指令来直接访问。

异常链接寄存器

异常链接寄存器保存首个异常返回地址。在AArchi64架构下，每个异常级别都有一个异常链接寄存器（ELR）

  • ELR_ELx（Exception Link Register）：用于保存进入异常时的程序计数器值。
  • SPSR_ELx（Saved Program Status Register）：保存异常发生时的程序状态。

保存处理状态寄存器

当进行异常处理时，处理器的处理状态会保存到保存处理状态寄存器（SavedProcess Status Register，（SPSR）里，这种寄存器非常类似于 ARMv7 架构中的 CPSR。当异常将要发生时，处理器会把处理状态寄存器（PSTATE）的值暂时保存到 SPSR 里；当异常处理完成并返回时，再把 SPSR 中的值恢复到处理器状态寄存器。

栈指针寄存器

ARM64 架构有不同的堆栈指针（SP_EL0, SP_EL1, SP_EL2, SP_EL3），它们表示不同特权级别下的堆栈指针。每一个异常等级都有专门的 SP_ELn 寄存器，比如当处理器运行在 EL1 时就选择 SP_EL1 寄存器作为栈指针（Stack Pointer，SP）寄仔器。

  • SP EL0：EL0 下的栈指针寄存器。
  • SP EL1：EL1 下的栈指针寄存器。
  • SP EL2：EL2 下的栈指针寄存器。
  • SP EL3：EL3 下的栈指针寄存器。

当处理器运行在比 EL0 高的异常等级时：

  • 处理器可以访问当前异常等级对应的栈指针寄存器 SP_ELn。
  • EL0 对应的栈指针寄存器 SP_ELO 可以当作临时寄存器，比如 Linux 内核就使用这种临时寄存器存放进程的 task struct 数据结构的指针。
  • 当运行在 ELO 时，处理器只能访问 SP_EL0 寄存器，而不能访问其他高等级的 SP 寄存器。

浮点和SIMD寄存器（V寄存器）

ARM64 也支持 SIMD（单指令多数据）和浮点运算，并且有 32 个 128 位的 V 寄存器，V0 到 V31。这些寄存器用于存储浮点数、SIMD 数据以及一些特殊的操作结果。

  • V0 到 V31：这些寄存器可以用于浮点运算，也可以用于 SIMD 运算，如单精度和双精度浮点运算。它们的大小为 128 位。

V 寄存器的分配方式：

  • S 寄存器：对应每个 V 寄存器的单精度浮点部分（S0 到 S31）。
  • D 寄存器：对应每个 V 寄存器的双精度浮点部分（D0 到 D31）。
  • Q 寄存器：对应每个 V 寄存器的 4 组 32 位寄存器部分。

系统寄存器

ARMv8 架构还定义了很多系统寄存器，可通过访问和设置这些系统寄存器来完成对处理器不同功能的配置。在 ARMv7 架构里，我们需要通过访问 CP15 协处理器来间接访问这些系统寄存器：而在 ARMv8 架构中没有协处理器，可直接访问系统寄存器。ARMv8 架构支持如下 7 大类的系统寄存器：

  • 通用系统控制寄存器
  • 调试寄存器
  • 性能监控寄存器
  • 活动监控寄存器
  • 统计扩展寄存器
  • RAS 寄存器
  • 通用定时器寄存器

系统寄存器支持不同异常等级下的访问，通常系统寄存器可使用“Reg_ELn”的方式来表示，示例如下。

  • Reg_EL1：处理器处于 EL1、EL2 以及 EL3 时可以访问该寄存器。
  • Reg_EL2：处理器处于 EL2 和 EL3 时可以访问该寄存器。

ARM64指令集

ARM64 指令集是 ARM 架构的 64 位版本，也称为 AArch64，是 ARM 公司设计的一种精简指令集计算（RISC）架构。它最初是为了满足移动设备和嵌入式系统对更高性能和更大内存寻址能力的需求而开发的，后来逐渐扩展到服务器、笔记本电脑等领域。ARM64 的核心特点是其高效性和低功耗，这使得它在智能手机、平板电脑、物联网设备以及数据中心中得到了广泛应用。

ARM64 支持 64 位寄存器和内存地址，能够访问更大的内存空间（理论上最多支持 2^64 字节的内存），同时它也兼容 32 位的 ARM 指令集（AArch32），允许运行 32 位应用程序。ARM64 提供了 31 个通用寄存器（64 位），比 32 位 ARM 架构的 16 个寄存器更多，这些寄存器可以用于存储数据、地址和临时计算结果。ARM64 的指令集是固定长度的，每条指令占 32 位，设计简洁，支持高效的流水线执行，同时还支持更多的数据处理指令和浮点运算指令。

ARM 汇编指令集可以根据指令的功能和操作类型分为以下几种主要类型：

类型 常见指令 说明
数据处理指令 ADD, SUB, AND, ORR, LSL, ASR 算术、逻辑、位移操作
数据传输指令 LDR, STR, LDM, STM 寄存器与内存之间的数据传输
分支与跳转指令 B, BL, BX, BEQ, BNE 控制程序流，条件/无条件跳转
状态访问与控制指令 MRS, MSR, CPS, SETEND 访问状态寄存器，控制标志设置
多媒体处理指令 VADD, VMUL, VLD, VST SIMD 和 NEON 向量运算指令
协处理器指令 MCR, MRC, CDP 协处理器通信
系统指令 SVC, BKPT, CLREX 系统级操作，如中断、调试等
同步指令 LDREX, STREX, DMB, DSB 多处理器环境中的同步与内存屏障指令
