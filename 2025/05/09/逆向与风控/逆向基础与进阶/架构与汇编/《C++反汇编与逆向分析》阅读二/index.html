在C++中，字符串本身就是数组，根据约定，该数组的最后一个数据统一使用0作为字符串结束符。在编译器下，为字符类型的数组赋值（初始化）其实是复制字符串的过程。

当数组作为参数时，数组的下标值被省略了。这是因为当数组作为函数形参时，函数参数中保存的是数组的首地址，这是一个指针变量。虽然参数是指针变量，但需要特别注意的是，实参数组名为常量值，而指针或形参数组为变量。使用sizeof（数组名）可以获取数组的大小，而对指针或者形参中保存的数组名使用sizeof只能得到当前平台的指针长度。

数组作为返回值

数组作为函数参数：

  • 编译器将数组的首地址以指针形式传递。
  • 数组的定义作用域在函数调用之外，调用前已存在，因此在函数中操作数组是安全的。

数组作为函数返回值：

  • 如果数组是局部变量，其内存空间位于当前函数的栈内。
  • 函数退出时，栈会平衡，局部数组的内存空间可能被后续函数调用覆盖，导致数据不稳定。
  • 这种不稳定性会导致返回结果不可靠，影响程序正确性。

局部数组作为返回值时，函数退出后其内存可能被破坏。

  • 使用动态分配的内存（如 malloc 或 new）返回数组，确保内存生命周期不受函数栈影响。
  • 将数组封装在结构体或类中返回，避免直接返回局部数组。

汇编代码：

由于数组buffer为局部变量，因此其所占内存空间的位置在栈空间内，生命周期随函数的退出而结束。而在函数结束后，将数组的首地址赋值到eax中作为返回值。虽然这个地址始终存在，但它是栈空间中的某段内存空间，其中的数据会在作用域切换时被新数据替换，因此返回局部变量的地址随时会产生错误。

下标寻址和指针寻址

访问数组的两种方法：

下标访问（数组名[标号]）：

  • 更为简便，是常用的访问方式。
  • 直接通过数组名（常量地址）和下标计算偏移量，访问目标数据。

指针访问（指针寻址）：

  • 需要先取出指针变量中存储的地址，再进行偏移计算。
  • 效率较低，因为多了一步取指针值的操作。

汇编代码：

下标访问因为数组名是常量地址，编译器可以直接计算偏移量，访问效率高。而指针访问，指针是变量，访问时需要先读取指针的值，再进行偏移计算，效率较低。

虽然使用指针寻址方式需要经过2次间接访问，效率较低，但其灵活性更强，可通过修改指针中保存的地址数据，访问其他内存中的数据，而数组下标在没有越界使用的情况下只能访问数组内的数据。

ary[n] 的地址 = ary 的首地址 + sizeof(type)*n

注意：

当下标值为变量时，编译器无法算出对应的地址，只能先进行地址偏移计算，再得出目标数据所在的地址。当下标值为表达式时，会先算出表达式的结果，然后将其作为下标值。如果表达式为常量计算，则编译过程中执行常量折叠，编译时提前计算出结果，其结果依然是常量，最后还是以常量作为下标，藉此寻址数组内元素。

数组下标寻址利用首地址加偏移量。数组的3种下标寻址方案同样也可以应用在指针寻址中。在编译器中，不会对数组的下标进行访问检查，使用数组时很容易出现越界访问的错误。

指针寻址：两次以上间接访问，第一次访问的值作为地址。

数组下标寻址：

  • 常量下标：直接基址加偏移，无二次寻址。
  • 变量下标：体现寻址公式，第一次访问下标值，第二次计算地址。

虽然二维字符数组和指针数组的寻址过程非常相似，但依然有一些不同。字符指针数组寻址后，得到的是数组成员内容，而二维字符数组寻址后得到的是数组中某个一维数组的首地址。如果代码中没有初始化操作，就需要分析如何寻址数据。

指针寻址方法

指针变量 += 数值⇔指针变量地址数据 +=（sizeof( 指针类型 ) * 数值）

核心要点：

  • 指针运算：需要知道数据类型以计算偏移量。
  • 多维数组的指针表示：
    • 将最高维看作一维数组，其后维度作为元素类型。
    • 例如，int ary[3][4][5] 可看作 int[4][5] ary[3]。
  • 正确语法：
    • 使用 int (*p)[4][5] = ary; 表示指向二维数组的指针。这种语法表示 p 是一个指向 int[4][5] 类型数组的指针。
    • 由于 C++ 语法不支持 int[4][5] *p 这种格式，因此需要使用括号明确优先级。
  • 与指针数组的区别：
    • int (*p)[4][5] 是指向二维数组的指针。
    • int *p[4][5] 是指针数组。

函数指针

函数指针的概念：

  • 函数指针是指向函数首地址的指针变量。
  • 通过 call 指令跳转到函数首地址执行函数代码。

函数指针的定义：

函数指针的定义与函数声明类似，由以下 4 部分组成：

  1. 返回值类型：函数返回的数据类型。
  2. 指针名称：函数指针变量的名称。
  3. 参数列表：函数的参数类型和数量。
  4. 调用约定：决定参数传递、返回值处理和栈平衡方式。

函数指针的类型：

  • 函数指针的类型由 返回值类型、参数信息 和 调用约定 共同决定。
  • 调用约定决定了函数调用过程中参数的传递方式、返回值处理方式以及栈平衡方式。

调用约定：

  • 如果没有特殊说明，函数指针的调用约定与 C++ 编译器的默认设置一致。
  • 常见的调用约定包括 __cdecl、__stdcall、__fastcall 等。

总结

函数指针的赋值和调用：

  • 赋值：将函数的首地址赋值给函数指针变量。
  • 调用：通过函数指针间接调用函数，需要先取出指针变量中保存的地址数据。

与直接函数调用的区别：

  • 直接调用函数时，编译器直接跳转到函数首地址执行代码。
  • 函数指针调用时，需要先读取指针变量中的地址，再跳转到该地址执行代码。

函数指针的特殊性：

  • 函数指针保存的是代码段内的地址信息，而不是数据区的地址。
  • 函数指针不支持地址偏移操作（如加法、减法），因为代码段的地址偏移没有意义。

函数指针的类型匹配：

  • 函数指针的类型由返回值类型、参数列表和调用约定决定。
  • 如果函数指针的参数和返回值为 void 类型，则只能存储相同类型的函数地址。
  • 如果函数指针带参数和返回值，必须确保类型匹配，否则无法正确传递参数、返回值和平衡栈顶。
