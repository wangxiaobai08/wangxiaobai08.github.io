关于递归的思想递归详解——让你真正明白递归的含义-CSDN博客这篇文章讲的比较透彻，遇到问题，首先思考：

问题本身是否可以通过层层拆解到最小粒度来得解。

如何化大为小？能否找到当前的问题和子问题的关系关联？

然后程序的主要思想归纳起来就是：

  1. 明确你这个函数想要干什么

高度抽象+返回类型和参数(即我们暂时不管这个函数是怎么实现的，先下定义即可)，在递归中就视为顺序执行，逻辑通顺即可

就像学霸一样，看似什么都没做，其实人家早就做好了

  1. 寻找递归结束条件

视情况分析,最小子问题—->是指递和归的转折点，应当和归过程视为一起

  1. 找出函数的等价关系式【比如：f(n) = n * f(n-1)。】

归的的节点—>视为回调函数

21. 合并两个有序链表 - 力扣（LeetCode）

题目描述：将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

思考：新的 升序 链表是不断由小的升序 链表合并而来

1.链表的关键在于寻找下一个节点，故这个函数的目的就是寻找当前节点的下一个节点

2.结束条件就是节点的值为null

3.函数关系：

3.1 a(n)=min(a(n-1)+b(n))

3.2 b(n)=min(b(n-1)+a(n))

390. 消除游戏 - 力扣（LeetCode）

题目描述：列表 arr 由在范围 [1, n] 中的所有整数组成，并按严格递增排序。请你对 arr 应用下述算法：

  • 从左到右，删除第一个数字，然后每隔一个数字删除一个，直到到达列表末尾。
  • 重复上面的步骤，但这次是从右到左。也就是，删除最右侧的数字，然后剩下的数字每隔一个删除一个。
  • 不断重复这两步，从左到右和从右到左交替进行，直到只剩下一个数字。

给你整数 n ，返回 arr 最后剩下的数字。

提示：当前的问题和子问题中序列下标的映射关系。(就是找规律)

思考：

从解释中就可以看到：大列表的结果是由处理后的小列表决定的。我们要做的就是找到：1.怎么处理的？——>整体处理过程

2.处理后的小列表与大列表的关系(规律)？—–>大小问题关系的编写

1.函数目的：找到列表中的最后一个元素

2.返回条件：n==1—->最小子问题

3.关系：

先找规律：

1：左到右 操作的情况 :
操作前 : 1,2,3,4,5… (序列 0)
操作后 : 2,4,6,… (序列 1)
可以发现操作后的序列可以看成是 : 2 * (1,2,3… (序列 2) )

2.右到左 操作的情况：
因为当序列长度为 奇数 和 偶数的情况 , 会对开头的 1 是否删除有影响 , 那么我们分类讨论 ：

2.1当 n 为奇数时 (以 n 等于 7 为例进行说明) :
原序列 : 1 2 3 4 5 6 7 (序列 0)
操作后 : 2 4 6 (序列 1)
那么接下去的一步 , 对 (序列 1) 的操作同样可以转为 对 (序列 2) 的操作 , 选出的数字 * 2 即可 ((序列 2) = {1，2，3})
2.2当 n 为偶数时 (以 n 等于 8 为例进行说明) :
原序列 : 1 2 3 4 5 6 7 8 (序列 0)
操作后 : 1 3 5 7 (序列 1)
那么接下去的一步 , 对 (序列 1) 的操作同样可以转为 对 (序列 2) 的操作 , 选出的数字 * 2 - 1 即可 ((序列 2) = {1，2，3，4})

从左到右：f(n)=2*f(n/2)

从右到左:

奇数：f(n)=2*f(n/2)

偶数：f(n)=2*f(n/2)-1

编写代码过程中发现，整体处理过程第一个是从左到右，往后是从右到左，递归流程不好设置，可以把递归代码抽离出来(辅助函数)，初始在主函数里

这题一开始用快慢指针做的，但浏览了官方递归的解法，虽然递归不是最好的选择，但巧妙的体现了递归的思想。直接上官方代码

递归为我们提供了一种优雅的方式来方向遍历节点。

如果使用递归反向迭代节点，同时使用递归函数外的变量向前迭代，就可以判断链表是否为回文。更详细的解释就是：

recursivelyCheck 函数是一个递归函数，用于检查链表是否是回文。

  1. 递归调用 recursivelyCheck(currentNode->next)：
    • 递归调用的目标是逐个节点地从链表的尾部往头部比较，直到找到链表的终点（即 currentNode == nullptr）。这个递归调用实际上会遍历链表的所有节点，先遍历到链表的尾部，然后再从尾部开始返回，逐个节点地进行比较。
  2. if (!recursivelyCheck(currentNode->next)) { return false; } 的含义：
    • 当递归到达链表的终点时，递归函数会返回 true，然后递归逐层返回到每个节点，并进行节点值的比较。
    • 如果在某一层递归中，发现下一个节点的值不等于当前节点的值（通过 currentNode->val != frontPointer->val 进行比较），那么就意味着链表不是回文的。此时递归会返回 false，并且通过这一层的 if (!recursivelyCheck(currentNode->next)) { return false; } 语句将错误传递给上一层递归，最终返回 false。
  3. 为什么要使用递归的方式：
    • 递归的方式确保了链表的尾部和头部能够逐一进行比较，因为每次递归会先“进入”到链表的尾部（即通过 currentNode->next 向前递归），然后再逐步返回到头部，在回程的过程中进行比较。通过这种方式可以巧妙地实现对链表的回文检查。
  4. frontPointer 的作用：
    • frontPointer 是一个指向链表头部的指针，它在每次递归返回时向后移动（通过 frontPointer = frontPointer->next），这样就能够比较当前递归节点（currentNode->val）和链表头部节点的值。通过这种方式，链表的头部和尾部节点会一一进行比较。

可以看到利用了递归，先推(遍历)再归的过程中巧妙进行对比。

奇怪的是在generate_all中似乎顺序的递归了两次？这该如何理解这段核心代码：

首先尝试向 current 添加一个左括号 '('：

  • 更新 current 为 current + '('。
  • 递归调用 generate_all，进入下一层递归。
  • 递归返回后，通过 pop_back 撤销刚才的修改，回溯到上一状态。

然后尝试向 current 添加一个右括号 ')'：

  • 同样，更新 current 为 current + ')'，进入递归。
  • 递归返回后，再次通过 pop_back 撤销修改。

递归树分析

递归生成的是一棵二叉树，每个节点表示当前 current 的状态，每次递归分别尝试添加 '(' 和 ')'：

  • 根节点：current = ""（初始状态）。
  • 第一层：尝试加 '(' 或 ')'，即 current = "(" 或 current = ")"。
  • 第二层：继续尝试，每个分支再加 '(' 或 ')'。
  • 终止节点：当 current.size() == n 时，检查是否有效。

回溯

通过 pop_back()，每次递归返回时，撤销最近一次的修改，恢复到上一状态，以便尝试另一分支。这种机制确保可以遍历所有可能的组合。

整体递归流程

举例：n = 4（长度为 4 的括号组合）

  1. 初始：current = ""

  2. 加 ‘(‘：current = “(“

    继续递归，最终构造所有以 '(' 开头的组合。

  3. 回溯后加 ‘)’：current = “)”

    构造所有以 ')' 开头的组合。

  4. 每次到达长度 n 后检查有效性并返回结果。

可用看到这样有出现了一个核心的算法思想：回溯

回溯，向上推导。一个问题推导出多种可能，则选中其中一种可能继续推导。如果一条路到达尽头，则回到起点，选择另外的路径继续推导，直到所有可能被推导完毕。（另外的路径：含有与之前推导过的路径 不同的可能的路径）

如果知道深度优先搜索，回溯就好像每结束深搜中的一条路线，都会返回一个这条路线上的元素所组成的序列，当所有序列被 一 一 列出，回溯结束。

算法上回溯常被用递归来实现，我把它理解成“子状态不唯一的递归”（或者是“多分支的递归”）

回溯有剪枝的功能：每次剪掉 (输出) 符合条件的树枝，当回溯结束时（若问题有序：当遇到不符合条件的树枝时停下，一种搜索过程的优化）我们就获得了所有符合条件的树枝 (答案)。
( 剪掉的部分是末端顶点到离它最近的一个有分叉的点)
