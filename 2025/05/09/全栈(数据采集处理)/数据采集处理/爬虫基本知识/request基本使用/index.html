基本用法

请求与响应对象

任何时候进行了类似 requests.get() 的调用，你都在做两件主要的事情。其一，你在构建一个 Request 对象， 该对象将被发送到某个服务器请求或查询一些资源。其二，一旦 requests 得到一个从服务器返回的响应就会产生一个 Response 对象。该响应对象包含服务器返回的所有信息，也包含你原来创建的 Request 对象。

高级用法

会话对象

在 Web 开发中，cookie 用于存储客户端（浏览器）和服务器之间的会话信息，例如登录状态、用户设置等。当你访问网站时，服务器可能会向你的浏览器发送一个 cookie，浏览器会在后续的请求中将这个 cookie 发送回服务器。

通常，每次使用 requests.get() 或 requests.post() 发出请求时，这些请求都是独立的，它们不共享任何信息。也就是说，每次请求都没有记住上次请求的状态。

会话对象的作用：

  1. 跨请求保持 cookie：当你使用 requests.Session() 创建一个会话对象时，它会自动保存和管理会话期间的 cookie。这意味着，当你向同一个服务器发送多个请求时，前一个请求的 cookie 会被保存下来并在下一个请求中自动使用，从而模拟了在浏览器中的持续会话。

    例如，当你登录一个网站时，服务器可能会设置一个 session_id cookie。使用会话对象后，后续的请求会自动携带这个 session_id，使得服务器可以识别你仍然是同一个用户，而不需要重新登录。

  2. 连接池和性能提升：使用会话对象还能够通过 连接池 功能重用底层的 TCP 连接，减少了每次请求时重新建立连接的开销，从而提高性能，特别是在对同一主机进行多个请求时。

requests.request() 和 requests.Session().request()

requests.request() 方法和 requests.Session().request() 方法都可以用来发起 HTTP 请求，但它们在实现方式和使用场景上有所不同。虽然两者都调用了 Session 类的 request() 方法，但其差异主要体现在 会话管理 和 内部实现 上。

两者调用 Session 类的 request() 方法的区别?
  • **requests.request()**：
    • 是一个 顶级函数，调用时会临时创建一个新的 Session 对象，并在该对象上发起请求。每次调用 requests.request() 都是一次 独立的请求，请求之间没有任何共享状态（如 cookies 或连接池）。
    • 它并不保持会话状态，每次请求都没有连接池和 cookie 管理。
  • **requests.Session().request()**：
    • 这是 Session 类的 实例方法，会话对象用于 跨多个请求保持会话状态（例如自动携带 cookies、HTTP 头部、连接池复用等）。
    • 使用 Session() 对象，你可以在多个请求之间保持相同的会话信息，如登录的 cookies，不需要每次手动传递请求参数。。
