高级用法

会话对象

在 Web 开发中，cookie 用于存储客户端（浏览器）和服务器之间的会话信息，例如登录状态、用户设置等。当你访问网站时，服务器可能会向你的浏览器发送一个 cookie，浏览器会在后续的请求中将这个 cookie 发送回服务器。

通常，每次使用 requests.get() 或 requests.post() 发出请求时，这些请求都是独立的，它们不共享任何信息。也就是说，每次请求都没有记住上次请求的状态。

会话对象的作用：

  1. 跨请求保持 cookie：当你使用 requests.Session() 创建一个会话对象时，它会自动保存和管理会话期间的 cookie。这意味着，当你向同一个服务器发送多个请求时，前一个请求的 cookie 会被保存下来并在下一个请求中自动使用，从而模拟了在浏览器中的持续会话。

    例如，当你登录一个网站时，服务器可能会设置一个 session_id cookie。使用会话对象后，后续的请求会自动携带这个 session_id，使得服务器可以识别你仍然是同一个用户，而不需要重新登录。

  2. 连接池和性能提升：使用会话对象还能够通过 连接池 功能重用底层的 TCP 连接，减少了每次请求时重新建立连接的开销，从而提高性能，特别是在对同一主机进行多个请求时。

requests.request() 和 requests.Session().request()

requests.request() 方法和 requests.Session().request() 方法都可以用来发起 HTTP 请求，但它们在实现方式和使用场景上有所不同。虽然两者都调用了 Session 类的 request() 方法，但其差异主要体现在 会话管理 和 内部实现 上。

两者调用 Session 类的 request() 方法的区别?
  • **requests.request()**：
    • 是一个 顶级函数，调用时会临时创建一个新的 Session 对象，并在该对象上发起请求。每次调用 requests.request() 都是一次 独立的请求，请求之间没有任何共享状态（如 cookies 或连接池）。
    • 它并不保持会话状态，每次请求都没有连接池和 cookie 管理。
  • **requests.Session().request()**：
    • 这是 Session 类的 实例方法，会话对象用于 跨多个请求保持会话状态（例如自动携带 cookies、HTTP 头部、连接池复用等）。
    • 使用 Session() 对象，你可以在多个请求之间保持相同的会话信息，如登录的 cookies，不需要每次手动传递请求参数。。

Lxml

lxml - Processing XML and HTML with Python

lxml 是一个用于处理 XML 和 HTML 的 Python 库，它提供了对 XML 和 HTML 的高效解析和操作能力。

HTML 和 XML 之间的区别

XML 是一种面向数据的标记语言，专注于描述和传输结构化信息；它注重可扩展性和自定义，适合数据存储和交换。

HTML 是一种面向显示的标记语言，专注于内容的呈现和用户交互；它结构固定且宽松，适合网页设计。

特性 XML（可扩展标记语言） HTML（超文本标记语言）
目的 用于表示、存储和传输数据，强调数据的结构化和自定义。 用于展示网页内容，描述文档的外观和行为。
语法规则 严格：标签必须正确嵌套，属性值必须用引号，必须有根标签。 宽松：标签可以不完全闭合，不区分大小写，错误不会阻止文档渲染。
标签定义 用户可以自定义标签。 使用预定义的固定标签（如 <html>、<div>）。
数据格式 数据驱动：描述数据及其结构，通常无视觉呈现。 视觉驱动：用来定义文档的结构和视觉内容。
用途 数据交换（如配置文件、Web 服务接口）。 构建和展示网页内容。
可扩展性 完全支持：用户可以根据需求定义自己的标签和结构。 不支持扩展：只能使用 HTML 标准定义的标签。
显示功能 仅存储数据，无法直接定义数据如何显示。 专为显示内容设计，提供文本、图像、超链接等显示功能。
严格性 严格遵守规则，错误会导致解析失败。 宽容性强，小错误不会影响页面渲染。
案例用途 配置文件（如 .xml 文件）、数据交换（如 SOAP、RSS）。 网页设计、超媒体文档展示。
属性 属性仅用于提供额外信息，内容以文本为主。 属性用于控制页面的外观和行为（如 style、class）。
大小写敏感性 区分大小写：<Tag> 和 <tag> 是不同的标签。 不区分大小写：<Tag> 和 <tag> 是等价的。
解析工具 通常需要专用的解析器（如 lxml、ElementTree）。 浏览器原生支持，无需额外解析工具。
支持的标准 W3C XML 标准。 W3C HTML 和 HTML5 标准。
数据验证 支持数据验证（如通过 DTD 或 XML Schema 验证）。 不支持数据验证，仅为文档展示服务。
与 CSS 的集成 通常不直接集成 CSS，需结合 HTML 或其他技术实现样式。 天生与 CSS 集成，用于控制页面样式和布局。
与 JavaScript 的集成 通常作为数据源（如通过 AJAX 获取和解析 XML 数据）。 可以直接与 JavaScript 结合，实现交互式网页功能。
人类可读性 设计为机器和人类都易于理解，适合存储复杂的结构化数据。 面向人类设计，主要用于网页内容和交互的编写。

3.1 轴选择器（Axes）

轴 说明 示例
child:: 子节点（默认轴，可省略）。 child::child 等同于 child。
parent:: 父节点。 //child/parent::root
ancestor:: 祖先节点。 //child/ancestor::root
descendant:: 后代节点（不包括自身）。 //root/descendant::child
self:: 当前节点自身。 //child/self::child
following-sibling:: 当前节点的后续兄弟节点。 //child[@id='1']/following-sibling::child
