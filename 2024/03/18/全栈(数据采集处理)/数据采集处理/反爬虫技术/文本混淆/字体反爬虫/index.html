字体反爬虫

原理讲解

字体反爬虫是一种常见的反爬虫技术，它通过在网页中使用自定义字体或特殊字符来隐藏或混淆真实的文本内容，目的是使得爬虫难以准确获取网页上的信息。这种技术利用了字体文件的特殊性质，爬虫无法直接解析网页中的自定义字体，因此需要在爬虫脚本中额外处理字体文件的下载和解析。

  1. 字体替换： 网页会将页面中的文本替换成自定义字体，这种字体通常包含一些不可读的字符或是动态生成的字符。浏览器渲染时，会根据字体文件来显示这些字符，而爬虫在抓取页面内容时只能抓取到字符编码或是加密后的字体信息。
  2. 文字转换： 自定义字体文件中的每个字形（glyph）可能代表一个不同的字符，网页上的文本在渲染时会根据字体映射来转换，而爬虫无法直接获取到这些映射关系，导致无法提取正确的文本。
  3. 加密或混淆： 字体文件可能会将常规的文字字符转换为加密后的字符，这样在浏览器中查看时是可读的，但爬虫无法直接解析这些字符。

生活中的例子：

可以把这个过程想象成，你在写一篇文章并将其用特殊的符号和字符进行加密，只有特定的人（即你的浏览器）知道如何将这些符号还原成可读的文字。其他人（如爬虫）如果拿到这篇加密文章，就无法轻易读取出文章的内容。

实现技术：

  1. CSS 中使用字体： 网页通常使用 @font-face 规则来加载自定义字体文件。通过将自定义字体应用到网页上的文字，爬虫在抓取时无法直接获取到真实的文字内容。
  2. JavaScript 动态加载字体： 字体文件通常是通过 JavaScript 动态加载的，爬虫无法像浏览器那样执行 JavaScript，因此无法获取到字体文件，导致无法正确显示内容。

3. 通过字体映射来混淆文字

某些网站会用特定的字体映射技术，网页中的字符（例如数字和字母）在渲染时会被替换为其他的图形字符，爬虫抓取到的内容会显示为乱码或图形，而不是文字。

如何应对字体反爬虫：

  1. 解析字体文件： 爬虫可以通过下载并解析网页加载的字体文件来逆向还原其中的文字。常见的字体文件格式有 woff, woff2, ttf, otf 等。
    • 下载字体文件后，可以使用字体解析工具（如 FontForge, ttf2eot）来查看字体映射。
    • 使用 OCR（光学字符识别）技术，分析字体渲染的图形，并尝试识别其中的字符。
  2. 使用浏览器模拟： 通过工具如 selenium 来模拟浏览器行为，加载字体文件并渲染页面，从而获取正常显示的文本。
  3. 使用 CSS 提取技术： 尽管字体反爬虫技术可以增加爬虫的难度，但有些网站并不会完全依赖字体反爬虫。在这种情况下，爬虫可以通过分析页面结构和 CSS 样式来提取实际的内容。

字体文件 WOFF

WOFF（Web Open Font Format）是一种专为网页设计的字体文件格式，常用于在网页中嵌入自定义字体。它是基于 OpenType 和 TrueType 字体格式的，并对字体文件进行了压缩，以便更适合网页加载和传输。

WOFF文件格式

WOFF（Web Open Font Format，Web开放字体格式）是一种网页所采用的字体格式标准。本质上基于SFNT字体（如TrueType），所以它具备TrueType的字体结构，我们只需要了解TrueType字体的相关知识即可。TrueType字体是苹果公司与微软公司联合开发的一种计算机轮廓字体，TrueType字体中的每个字形由网格上的一系列点描述，点是字体中的最小单位，字形与点的关系如图所示。

字体文件中不仅包含字形数据和点信息，还包括字符到字形映射、字体标题、命名和水平指标等，这些信息存在对应的表中，所以我们也可以认为TrueType字体文件由一系列的表组成，其中常用的表及其作用如图所示。

只有起止坐标和点坐标数据完全相同的字形，描述的才是相同字符。
