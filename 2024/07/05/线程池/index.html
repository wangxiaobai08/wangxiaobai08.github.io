线程与多线程机制

线程在Android中是一个很重要的概念，从用途上来说，线程分为主线程和子线程，主线程主要处理和界面相关的事情，而子线程则往往用于执行耗时操作。由于Android的特性，如果在主线程中执行耗时操作那么就会导致程序无法及时地响应，因此耗时操作必须放在子线程中去执行。

主线程和子线程

主线程（UI 线程）

主线程是进程所拥有的默认线程，在 Java 和 Android 中，一个进程默认只有一个主线程。主线程的主要职责是

  • 处理与界面交互相关的逻辑，如 UI 更新、事件响应等。
  • 运行四大组件（Activity、Service、BroadcastReceiver、ContentProvider）并处理它们与用户的交互。

其必须保持高响应速度，以确保用户界面的流畅性。如果在主线程中执行耗时操作，会导致界面卡顿，甚至触发 ANR（Application Not Responding）。

注意：从 Android 3.0 开始，系统强制要求网络访问必须在子线程中进行，否则会抛出 NetworkOnMainThreadException 异常。

子线程（工作线程）

除了主线程以外的线程都是子线程。其主要职责是执行耗时任务，如网络请求、I/O 操作、数据库访问、复杂计算等。

子线程不会阻塞主线程，确保主线程的高响应速度。子线程的执行结果通常需要通过 Handler 或 AsyncTask 等机制传递到主线程，以更新 UI。

使用场景主要集中在：

  • 网络请求（如 HTTP 请求、Socket 通信）。
  • 文件读写（如加载大文件、保存数据）。
  • 数据库操作（如查询、插入、更新）。
  • 复杂计算（如图像处理、数据分析）。

主线程与子线程的协作

耗时操作必须放在子线程：

主线程中执行耗时操作会导致界面卡顿，甚至触发 ANR。子线程负责执行耗时任务，确保主线程的流畅性。

子线程与主线程的通信：

子线程执行完任务后，通常需要将结果传递到主线程以更新 UI。常用的通信机制包括：

  • Handler：通过消息队列实现线程间通信。
  • AsyncTask：封装了线程池和 Handler，方便在子线程中执行任务并更新 UI。
  • LiveData 和 ViewModel：基于观察者模式，实现数据驱动的 UI 更新。
  • RxJava：通过响应式编程实现线程切换和数据传递。

Native 线程：由 C/C++ 代码创建，分为可以访问 Java 代码和不能访问 Java 代码的线程。

Android 中的线程模型

沿用 Java 的线程模型：Android 的线程模型基于 Java，分为主线程和子线程。

主线程的特殊性：主线程是唯一可以直接更新 UI 的线程。主线程的运行机制基于 Looper 和 Handler，通过消息循环处理事件。

子线程的多样性：子线程可以通过多种形式实现，如 Thread、AsyncTask、HandlerThread、IntentService 等。子线程的执行通常依赖于线程池，以提高线程的复用率和系统性能。

Android中的线程形态

AsyncTask

AsyncTask是一种轻量级的异步任务类，它可以在线程池中执行后台任务，然后把执行的进度和最终结果传递给主线程并在主线程中更新UI。从实现上来说，AsyncTask封装了Thread和Handler，通过AsyncTask可以更加方便地执行后台任务以及在主线程中访问UI，但是AsyncTask并不适合进行特别耗时的后台任务，对于特别耗时的任务来说，建议使用线程池。

AsyncTask是一个抽象的泛型类，它提供了Params、Progress和Result这三个泛型参数，其中Params表示参数的类型，Progress表示后台任务的执行进度的类型，而Result则表示后台任务的返回结果的类型，如果AsyncTask确实不需要传递具体的参数，那么这三个泛型参数可以用Void来代替。AsyncTask这个类的声明如下所示。

方法 调用时机 执行线程 作用
onPreExecute() 在异步任务执行之前调用 主线程（UI 线程） 用于任务开始前的准备工作，例如显示进度条、初始化 UI 等。
doInBackground() 在 onPreExecute() 执行完毕后调用 子线程（后台线程） 执行耗时任务，例如网络请求、文件读写等。通过 publishProgress() 更新进度。
onProgressUpdate() 在 doInBackground() 中调用 publishProgress() 时触发 主线程（UI 线程） 更新任务进度，例如更新进度条、显示进度百分比等。
onPostExecute() 在 doInBackground() 执行完毕后调用 主线程（UI 线程） 处理任务结果，例如隐藏进度条、显示结果等。
onCancelled() 当异步任务被取消时调用 主线程（UI 线程） 处理任务被取消的情况，例如清理资源、更新 UI 等。

补充说明

执行顺序：

onPreExecute() → doInBackground() → onProgressUpdate()（可选） → onPostExecute()。

如果任务被取消，onCancelled() 会被调用，而 onPostExecute() 不会执行。

注意事项：

生命周期问题：AsyncTask 的生命周期与 Activity 或 Fragment 无关，可能导致内存泄漏。

版本差异：从 Android 4.0 开始，AsyncTask 默认是串行执行的，可通过 executeOnExecutor(AsyncTask.THREAD_POOL_EXECUTOR) 实现并行执行。

弃用：AsyncTask 已被标记为弃用，建议使用 LiveData、ViewModel、RxJava 或 Coroutine 替代。

executeOnExecutor

在指定的执行器（Executor）上执行异步任务。并管理任务状态，确保任务只能执行一次。调用 onPreExecute() 方法，执行任务开始前的准备工作。

sDefaultExecutor实际上是一个串行的线程池，一个进程中所有的AsyncTask全部在这个串行的线程池中排队执行，这个排队执行的过程后面会再进行分析。在executeOnExecutor方法中，AsyncTask的onPreExecute方法最先执行，然后线程池开始执行。

sDefaultExecutor

SerialExecutor 是 AsyncTask 的默认执行器，用于按顺序执行任务。它通过任务队列和线程池的结合，确保任务按 FIFO 顺序执行，同时利用线程池提高执行效率。

HandlerThread

HandlerThread 是 Android 提供的一个特殊线程类，它继承了 Thread，并在内部实现了消息队列和消息循环机制，使得可以在该线程中使用 Handler。以下是关于 HandlerThread 的详细解析：

HandlerThread 的核心特点

  • **继承自 Thread**：HandlerThread 是一个线程类，具备普通线程的所有特性。
  • 内部实现消息队列：通过 Looper.prepare() 和 Looper.loop() 创建消息队列并开启消息循环。
  • **支持 Handler**：可以在 HandlerThread 中创建 Handler，并通过 Handler 发送消息来执行任务。
  • 无限循环：run() 方法是一个无限循环，直到调用 quit() 或 quitSafely() 方法终止线程。

HandlerThread 与普通 Thread 的区别

特性 HandlerThread 普通 Thread

线程间通信：通过 Handler 发送消息到 HandlerThread，实现线程间通信。

IntentService

IntentService 是 Android 提供的一种特殊的 Service，用于在后台执行耗时任务。它的特点是任务执行完毕后会自动停止，并且由于它是 Service，其优先级比普通线程高，不容易被系统杀死。然而，在 Android 8.0 及以上版本中，建议使用 JobIntentService 或 WorkManager 替代 IntentService。

IntentService 的特点

自动停止：IntentService 在执行完任务后会自动停止，无需手动调用 stopSelf()。

任务队列：如果多次启动 IntentService，任务会按顺序执行（通过 HandlerThread 实现）。

高优先级：由于 IntentService 是 Service，其优先级比普通线程高，不容易被系统杀死。

子线程执行：onHandleIntent() 方法在子线程中执行，适合执行耗时任务。

IntentService 的源码分析

IntentService 的核心实现如下：

  • **HandlerThread**：用于在子线程中执行任务。
  • **ServiceHandler**：处理任务消息，并调用 onHandleIntent() 方法。
  • 自动停止：任务执行完毕后调用 stopSelf() 停止 Service。

IntentService 的生命周期

(1) onCreate()

当 IntentService 第一次启动时调用。创建一个 HandlerThread，用于在子线程中执行任务。并使用 HandlerThread 的 Looper 构造一个 Handler 对象（mServiceHandler）。

(2) onStartCommand()

每次启动 IntentService 时调用。调用 onStart() 方法，将任务 Intent 封装为消息并发送到 HandlerThread。

(3) onStart()

在 onStartCommand() 中调用。将任务 Intent 封装为 Message，并通过 mServiceHandler 发送到 HandlerThread。

(4) onHandleIntent()

在 HandlerThread 中处理消息时调用。执行具体的后台任务，任务逻辑由开发者实现。任务执行完毕后，调用 stopSelf(int startId) 尝试停止服务。

(5) stopSelf(int startId)

在 onHandleIntent() 执行完毕后调用。尝试停止服务，但会等待所有消息处理完毕后才真正停止。使用 startId 判断是否所有任务都已完成。

任务执行机制

(1) 任务队列

IntentService 内部通过 HandlerThread 实现任务队列，任务按 FIFO（先进先出）顺序执行。每次启动 IntentService，任务 Intent 会被封装为 Message 并发送到 HandlerThread 的消息队列中。

(2) 顺序执行

HandlerThread 的 Looper 会顺序处理消息，因此 IntentService 也是顺序执行任务的。如果有多个后台任务，这些任务会按照启动顺序排队执行。

(3) 自动停止

当 onHandleIntent() 执行完毕后，IntentService 会调用 stopSelf(int startId) 尝试停止服务。stopSelf(int startId) 会等待所有消息处理完毕后才真正停止服务。

注意事项

任务顺序执行：IntentService 的任务是按顺序执行的，如果任务之间没有依赖关系，可以考虑使用 JobScheduler 或 WorkManager。

Android 8.0 及以上版本：在 Android 8.0（API 26）及以上版本中，startService() 会抛出异常，建议使用 JobIntentService 或 WorkManager。

任务耗时限制：如果任务耗时过长，可能会导致 ANR（应用无响应），建议将任务拆分为多个小任务。

Android中的线程池

在操作系统中，线程是操作系统调度的最小单元，同时线程又是一种受限的系统资源，即线程不可能无限制地产生，并且线程的创建和销毁都会有相应的开销。当系统中存在大量的线程时，系统会通过时间片轮转的方式调度每个线程，因此线程不可能做到绝对的并行，除非线程数量小于等于CPU的核心数，一般来说这是不可能的。试想一下，如果在一个进程中频繁地创建和销毁线程，这显然不是高效的做法。正确的做法是采用线程池，一个线程池中会缓存一定数量的线程，通过线程池就可以避免因为频繁创建和销毁线程所带来的系统开销。Android中的线程池来源于Java，主要是通过Executor来派生特定类型的线程池

JobScheduler 和 WorkManager

JobScheduler

JobScheduler 是 Android 5.0（API 21）引入的框架，主要用于在特定条件下执行后台任务。它允许开发者设定任务执行的条件，例如设备充电时、网络连接时或设备空闲时。JobScheduler 会自动管理任务的调度，确保任务在满足条件时执行，同时尽量减少对电池的影响。它适合处理一些需要延迟执行或条件触发的任务，比如数据同步或日志上传。

WorkManager

WorkManager 是 Android Jetpack 的一部分，旨在提供一种更兼容和可靠的方式来执行后台任务。它兼容 Android 4.0（API 14）及以上版本，并能够在不同设备状态下（包括设备重启后）保证任务的执行。WorkManager 结合了 JobScheduler、AlarmManager 和 BroadcastReceiver 的优点，支持一次性任务和周期性任务。它还提供了链式任务、任务约束（如网络连接、充电状态）等功能，适合处理需要可靠执行的后台任务，例如数据备份或定期同步。

RxJava

参考：彻底搞清楚 RxJava 是什么东西-腾讯云开发者社区-腾讯云

RxJava 是一个基于观察者模式的异步编程库，主要用于简化复杂的异步任务和事件流处理。它通过提供丰富的操作符，允许开发者以声明式的方式处理数据流和事件流。RxJava 的核心概念是 Observable（被观察者）和 Observer（观察者），Observable 发出数据流，Observer 订阅并处理这些数据。RxJava 支持线程切换、错误处理、背压控制等功能，适合处理网络请求、数据库操作、UI 事件等异步场景。由于其强大的灵活性和可组合性，RxJava 在 Android 开发中被广泛用于管理复杂的异步逻辑。
