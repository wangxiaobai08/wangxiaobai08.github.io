JS的Hook技术

原理与作用

1. 作用域的基本概念

  • 全局作用域：在JavaScript中，全局作用域是整个程序中都可以访问的变量和函数。全局作用域中的变量和函数可以在任何地方使用。【如果在全局作用域中定义一个函数并对其进行Hook，则所有调用该函数的地方都将受到影响。这样，Hook可以用于修改全局作用域中的函数行为。】
  • 方法作用域（局部作用域）：指在函数内部定义的变量和函数只能在该函数内部访问，外部无法访问这些变量。【如果在一个函数内部对另一个函数进行Hook，则只会影响该函数内部的调用。在该函数外部，原函数的行为保持不变。】

2. Hook的基本概念

Hook是一种技术，允许开发者拦截或修改特定功能的行为。它可以应用于函数、对象方法、DOM事件等，以改变它们的默认行为。通过Hook，开发者可以在原始逻辑执行之前或之后插入自定义逻辑。

3. Hook与作用域的关系

闭包与Hook：
当使用Hook修改函数或对象方法时，闭包可以用于保持对原始上下文和变量的访问。Hook函数通过闭包能够访问其创建时的作用域，从而操作和修改状态。

原型链 Hook

原型链 Hook 是一种通过修改对象的原型来拦截和修改对象方法的技术。它通常用于改变类实例或内置对象的行为，而不需要直接修改原始对象。这种方式在 JavaScript 中非常灵活，因为 JavaScript 的对象是基于原型链的。

实现原型链 Hook

通过修改对象的原型，可以动态地添加、重写或删除方法。这种方法可以用于改变现有对象的行为，从而达到 Hook 的目的。

Proxy 对象 Hook

Proxy 是 ES6 引入的一种新的对象类型，它可以用来创建一个对象的代理，允许开发者拦截和定制基本操作（如属性查找、赋值、枚举、函数调用等）。通过 Proxy 对象，可以实现对原始对象的更细粒度控制，这为 Hook 提供了一种强大而灵活的方式。

Proxy的基本语法

Proxy 的构造函数接收两个参数：

  1. target：被代理的目标对象。
  2. handler：一个对象，其属性是拦截操作的方法。

代理的常用拦截器

在 handler 对象中，可以定义多个拦截器（陷阱），这些拦截器是函数，用于自定义对目标对象的基本操作。常用的拦截器包括：

  • get(target, property, receiver)：拦截对属性的访问。
  • set(target, property, value, receiver)：拦截对属性的赋值。
  • apply(target, thisArg, argumentsList)：拦截对函数的调用。
  • construct(target, argumentsList, newTarget)：拦截对构造函数的调用。

XHR的Hook

XHR（XMLHttpRequest）Hook 是一种技术，允许开发者拦截和修改 XMLHttpRequest 对象的行为。这种技术常用于监控、修改或增强通过 AJAX 发出的请求和响应，以实现功能如数据监控、请求伪造、响应修改等。

实现方式

XHR Hook 通常通过以下几种方式实现：

重写原型方法：
通过修改 XMLHttpRequest 的原型方法，例如 open()、send() 和 setRequestHeader()，来拦截请求和响应。

使用代理（Proxy）：
使用 JavaScript 的 Proxy 对象可以更精细化地控制对 XMLHttpRequest 实例的操作。

应用场景
  • 监控网络请求：开发者可以监控发送的请求和收到的响应，便于调试和分析。
  • 伪造请求：可以伪造请求以测试应用程序的反应。
  • 修改响应数据：在不修改后端代码的情况下，修改返回的数据以满足特定需求。
  • 防止敏感信息泄露：可以对发送的数据进行加密或屏蔽敏感信息。

Hook的时机

在JavaScript的Hook技术中，注入Hook的时机对于有效性和功能性至关重要。以下是关于Hook注入时机的详细分析和说明：

1. 控制台注入的Hook

  • 问题：当在浏览器控制台中手动注入Hook时，一旦网页刷新，所有注入的Hook都会失效。这是因为在页面加载时，所有的脚本都会被重新执行，之前注入的代码不会被保留。
  • 解决方案：为了确保Hook在页面加载时生效，可以在浏览器的开发者工具中设置断点（例如在第一个JavaScript文件的第一行）。这样可以在执行代码之前手动注入Hook，确保其在最初的执行上下文中有效。

2. 利用Fetch的替换响应注入Hook

  • 优点：使用Fetch API的替换响应（例如，利用XMLHttpRequest的重写）进行Hook注入，时机通常比较靠前。这种方法可以在网络请求发出之前进行拦截和修改，因此能够更好地控制请求和响应的行为。

3. 油猴脚本（Tampermonkey）

  • 问题：虽然油猴脚本可以在页面加载时执行自定义JavaScript，但由于脚本执行的顺序和时机可能不稳定，因此不建议使用它来进行重要的Hook操作。特别是在某些情况下，油猴脚本的执行可能晚于页面中关键JavaScript的加载，这样会导致Hook失效。
  • 建议：如果需要进行页面的深度定制或监控，可以考虑使用浏览器扩展程序，这些扩展允许更早地执行代码，确保在页面加载的第一时间就能够注入Hook。
