JNI机制分析

关于android中的Jni机制，在前面的文章中都或多或少介绍过。

比如在初识JVM - The Peak Tower我们谈到JNI的基本概念【JNI作为连接Java世界和C/C++世界的桥梁】和调用大致过程还有java层调用的本地代码在本地C实现的函数命名规则,

在android系统启动流程 - The Peak Tower中谈到了Android系统在启动启动过程中，先启动Kernel创建init进程，紧接着由init进程fork第一个横穿Java和C/C++的进程，即Zygote进程。Zygote启动过程中会AndroidRuntime.cpp中的startVm创建虚拟机，VM创建完成后，紧接着调用startReg完成虚拟机中的JNI方法注册。接着在register_jni_procs(gRegJNI, NELEM(gRegJNI), env)方法中循环调用gRegJNI数组成员所对应的方法。而gRegJNI数组，有100多个成员变量，每个成员都代表一个类文件的jni映射，其中REG_JNI是一个宏定义，在Zygote中介绍过，该宏的作用就是调用相应的方法。

在So加载和链接(linker) - The Peak Tower中我们介绍了用户定义的so如何被加载运行但一笔带过了FindSymbol调用JNI_Load

今天这篇文章将着重了解JNI的底层实现原理。

JNI原理分析

JNI注册

注册时机

  • Android系统启动过程中Zygote注册，可通过查询AndroidRuntime.cpp中的gRegJNI，看看是否存在对应的register方法；
  • 调用System.loadLibrary()方式注册。

注册方法

1. 静态注册

静态注册是通过在 Java 类的静态代码块中调用本地方法进行注册，并且通常使用 JNI_OnLoad 方法进行初始化。

静态注册过程：

  1. 在 Java 层声明 native 方法：

    • 使用 native 关键字声明一个本地方法，表示该方法将由 C/C++ 层实现。

  2. 在本地代码中实现 native 方法：

    • 本地代码中实现与 Java 层 native 方法相对应的函数。函数名根据 JNI 命名规范进行命名。

  3. 在本地代码中注册方法：

    • 在 JNI_OnLoad 函数中，通过 RegisterNatives 函数静态注册本地方法。JNI_OnLoad 是在本地库加载时调用的一个特殊函数。

优点：

  • 简洁性：方法注册和本地代码绑定在 JNI_OnLoad 中一次性完成，结构较为简单。
  • 性能：静态注册是通过本地库加载时一次性注册所有方法，性能上相对较好。

缺点：

  • 不够灵活：方法必须在本地库加载时进行注册，动态修改注册内容比较困难。
  • 不能在运行时修改或增加本地方法，只能在库加载时一次性注册。

2. 动态注册

动态注册是通过 Java 层的某个方法手动调用 RegisterNatives 来注册本地方法。这通常是用来在程序运行时决定是否注册特定的本地方法。

动态注册过程：

  1. 在 Java 层声明 native 方法：

    • 与静态注册一样，在 Java 层声明本地方法。

  2. 在本地代码中实现 native 方法：

    • 本地代码中实现对应的 native 方法。

  3. 在 Java 层手动加载库和注册方法：

    • 使用 System.loadLibrary 加载本地库。
    • 在 Java 类的静态初始化代码中调用 registerNativeMethods 进行动态注册。

  4. 在本地代码中实现方法注册：

    • 本地代码中实现 registerNativeMethods 函数，手动注册 native 方法。

优点：

  • 灵活性：方法可以在运行时注册，适用于根据条件选择性注册方法。
  • 可以在多个地方调用：可以在应用运行过程中根据需要多次注册不同的本地方法。

缺点：

  • 复杂性：相较于静态注册，动态注册需要手动调用注册函数，增加了代码复杂度。
  • 性能：每次注册都需要手动调用 RegisterNatives，如果频繁调用，可能对性能产生影响。

静态注册与动态注册的对比

特性 静态注册 动态注册
注册时机 本地库加载时（通过 JNI_OnLoad） 在 Java 代码中手动注册（通过 registerNativeMethods）
方法注册方式 一次性注册所有方法 可以根据需要在运行时动态注册特定方法
灵活性 低，方法必须在加载时全部注册 高，支持运行时注册和修改
性能 较好，注册一次，性能开销较小 可能稍差，如果频繁注册，可能增加开销
使用场景 一般用于方法数量固定、不变的情况 适用于需要根据条件注册不同方法的情况

注意：方法的参数是JNIEnv指针，其中JNIEnv是重点，它代表Native世界中Java环境的代表，通过JNIEnv* 指针可以在Native世界访问Java世界的代码，进行操作，它只在创建它的线程中有效，不能跨线程传递。

JNINativeMethod 是 Android JNI 中用于描述 Java 层方法和本地 C/C++ 实现之间映射关系的结构体。

  • **name**：Java方法的名称，例如 startSensorService。

  • signature方法的签名，用于标识方法的参数和返回值类型。这里 “()V”

jniRegisterNativeMethods是 Android 提供的工具函数，用于将 JNINativeMethod

中定义的本地方法注册到指定的 Java 类。

    • NELEM(gMethods)：计算数组元素个数，用于告知方法数量。
  • 作用： 将本地方法和 Java 方法绑定，Java 层调用 startSensorService 或 startHidlServices 方法时，会执行对应的 C/C++ 实现函数。

签名格式规则

  1. 基本数据类型
    • 单个字符表示类型（如 I 表示 int，D 表示 double）。
  2. 引用类型
    • 格式为：L<类全限定名>;。
    • 例如，java.lang.String 的签名为 Ljava/lang/String;。
  3. 数组类型
    • 格式为：[<元素类型>。
    • 例如，int[] 的签名为 [I，String[] 的签名为 [Ljava/lang/String;。
  4. 方法签名
    • 格式为：(<参数签名列表>)<返回值签名>。
    • 例如：
      • int add(int a, int b) 的签名为：(II)I。
      • void print(String message) 的签名为：(Ljava/lang/String;)V。
